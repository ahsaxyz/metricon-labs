## Contributing

Metricon Labs welcomes contributions from developers interested in privacy infrastructure and Solana protocol design.

### Ways to Contribute

- Submit bug reports or improvement suggestions
- Improve documentation
- Propose architectural improvements
- Submit pull requests for features or fixes

### Development Workflow

1. Fork the repository
   
2. Create a feature branch

```bash
git checkout -b feature/your-feature-name
```

3. Commit your changes

```bash
git commit -m "Add feature or improvement"
```

4. Push the branch

```bash
git push origin feature/your-feature-name
```

5. Open a Pull Request

### Contribution Guidelines

- Keep changes focused and well documented
- Follow existing project structure and conventions
- Include clear commit messages
- Avoid introducing unnecessary dependencies

---

Metricon aims to build open infrastructure for privacy-conscious execution on Solana, and thoughtful contributions are welcome.
