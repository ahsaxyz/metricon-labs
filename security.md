## Security

Metricon is being developed with security as a primary design consideration.

All vault logic, transaction routing mechanisms, and execution flows are designed to maintain verifiable on-chain behavior while minimizing unnecessary exposure of user activity patterns.

### Security Principles

- **Non-custodial architecture** — user assets remain under programmatic control rather than centralized custody.
- **Transparent smart contract logic** — all on-chain components are publicly auditable.
- **Deterministic vault execution** — program-controlled vault accounts enforce execution rules.
- **Separation of concerns** — frontend, relayer infrastructure, and on-chain logic are architecturally isolated.
- **Gradual deployment strategy** — components will be deployed and tested incrementally.

### Responsible Disclosure

If you discover a security vulnerability or potential exploit, please report it privately.

Contact:

**security@metriconlabs.com**

Please do not publicly disclose vulnerabilities until they have been reviewed and resolved.
